All Halcon configuration is managed through the `.env` file in the project root. Laravel reads this file at runtime, so changes take effect immediately without redeploying code — just restart the server if you change a cached value.

<Note>
  If `.env` does not exist, copy it from the example file: `cp .env.example .env`.
  Then generate an application key with `php artisan key:generate`.
</Note>

---

## Core application variables

| Variable | Description | Example |
|---|---|---|
| `APP_NAME` | The name shown in the browser tab and notifications | `Halcon` |
| `APP_ENV` | Runtime environment. Use `local` for development, `production` for live deployments | `local` |
| `APP_KEY` | 32-character encryption key. Generated by `php artisan key:generate` | `base64:...` |
| `APP_DEBUG` | Show detailed error pages. Set to `false` in production | `true` |
| `APP_URL` | The base URL of the application | `http://localhost` |
| `APP_LOCALE` | Default locale for date formatting and translations | `en` |
| `APP_TIMEZONE` | PHP timezone for all date/time operations | `UTC` |

A minimal development `.env` looks like this:

```env
APP_NAME=Halcon
APP_ENV=local
APP_KEY=base64:GENERATED_BY_ARTISAN
APP_DEBUG=true
APP_URL=http://localhost
```

<Warning>
  Set `APP_DEBUG=false` and `APP_ENV=production` on any publicly accessible
  server. Debug mode exposes stack traces, environment variables, and internal
  paths to the browser.
</Warning>

## Database

Halcon supports SQLite and MySQL. SQLite is the easiest option for local development because it requires no database server. MySQL is recommended for production deployments with concurrent users.

<Tabs>
  <Tab title="SQLite">
    SQLite stores the entire database in a single file. No server to install or manage.

    1. Create the database file:

       ```bash
       touch database/database.sqlite
       ```

    2. Set the following variables in `.env`:

       ```env
       DB_CONNECTION=sqlite
       # Leave the variables below unset or remove them entirely.
       # Laravel will use database/database.sqlite by default.
       ```

    3. Run migrations:

       ```bash
       php artisan migrate
       ```

    <Tip>
      The `composer setup` script automatically creates `database/database.sqlite`
      if it does not exist, so you can skip step 1 when using that shortcut.
    </Tip>
  </Tab>
  <Tab title="MySQL">
    MySQL is recommended for production. Create a dedicated database and user before configuring Halcon.

    ```sql
    CREATE DATABASE halcon CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
    CREATE USER 'halcon_user'@'localhost' IDENTIFIED BY 'strong_password';
    GRANT ALL PRIVILEGES ON halcon.* TO 'halcon_user'@'localhost';
    FLUSH PRIVILEGES;
    ```

    Then configure `.env`:

    ```env
    DB_CONNECTION=mysql
    DB_HOST=127.0.0.1
    DB_PORT=3306
    DB_DATABASE=halcon
    DB_USERNAME=halcon_user
    DB_PASSWORD=strong_password
    ```

    Run migrations:

    ```bash
    php artisan migrate
    ```
  </Tab>
</Tabs>

## Storage

Halcon's Route team uploads photographic evidence (loading photos and delivery photos) through the admin dashboard. These files are stored in `storage/app/public` and must be accessible via a public URL.

Run the following command once during setup to create a symbolic link from `public/storage` to `storage/app/public`:

```bash
php artisan storage:link
```

After this, uploaded images are reachable at `http://your-app-url/storage/filename.jpg`.

<Warning>
  If you skip `php artisan storage:link`, delivery photographs will upload
  successfully but will return 404 errors when customers try to view them.
</Warning>

## Queue driver

Halcon uses Laravel queues for background jobs. The queue driver determines how queued jobs are processed.

| Driver | Use case |
|---|---|
| `sync` | Jobs run immediately in the same process. No worker needed. Good for development. |
| `database` | Jobs are stored in the `jobs` table and processed by a separate queue worker. Recommended for production. |

```env
# Development (default)
QUEUE_CONNECTION=sync

# Production
QUEUE_CONNECTION=database
```

When using the `database` driver, run the queue worker alongside the web server:

```bash
php artisan queue:listen --tries=1 --timeout=0
```

The `composer dev` script (described below) handles this automatically in development.

## Mail

Halcon uses Laravel's mail system for notifications. Configure SMTP credentials in `.env`:

```env
MAIL_MAILER=smtp
MAIL_HOST=smtp.example.com
MAIL_PORT=587
MAIL_USERNAME=your@email.com
MAIL_PASSWORD=your_password
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS=no-reply@example.com
MAIL_FROM_NAME="${APP_NAME}"
```

During development, set `MAIL_MAILER=log` to write all outgoing mail to the log file instead of sending it:

```env
MAIL_MAILER=log
```

## Development workflow

The `composer dev` script starts four processes concurrently using `npx concurrently`:

- `php artisan serve` — the web server on `http://localhost:8000`
- `php artisan queue:listen --tries=1 --timeout=0` — the queue worker
- `php artisan pail --timeout=0` — a real-time log viewer in the terminal
- `npm run dev` — the Vite dev server with hot module replacement

```bash
composer dev
```

This is the recommended way to run Halcon during active development. You do not need to run `php artisan serve` and `npm run dev` separately.

<Tip>
  Use `composer dev` instead of `php artisan serve` for day-to-day development.
  It gives you live asset reloading and a built-in log stream in the same terminal window.
</Tip>

## Application environment (`APP_ENV`)

The `APP_ENV` variable changes application behavior in several ways:

- `local` — debug mode is typically enabled, detailed error pages are shown, mailable previews are available in the browser
- `production` — errors are logged silently, caching is optimized, `APP_DEBUG` should be `false`

Before going live, run the optimization commands to cache routes, views, and configuration:

```bash
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

To clear all caches:

```bash
php artisan optimize:clear
```
